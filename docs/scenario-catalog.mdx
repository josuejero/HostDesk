# Scenario Catalog

HostDesk ships with 9 synthetic Microsoft cloud sales-ops scenarios. Each record is defined in `data/scenario-catalog.json` and combines account context, activity history, playbook matches, scoring, and CRM hygiene review state.

## Hero demos

- **MSP evaluating AVD cost control**
  - Multi-tenant MSP motion focused on Azure Virtual Desktop cost visibility and margin protection.
- **Contractor fleet for Windows 365**
  - New inbound lead for secure contractor onboarding and BYOD-safe Cloud PC access.
- **Intune compliance rollout**
  - Active compliance and app-deployment motion driven by audit pressure.

## Additional states covered

- **Research needed**
  - Citrix migration record missing owner and dated next step.
- **Stale**
  - Healthcare shared-desktop account with overdue cadence.
- **Meeting booked**
  - Education lab-access motion that progressed from outreach to a confirmed session.
- **Handoff ready**
  - MSP multi-workload record ready for AE packaging.
- **Nurture**
  - Budget-blocked desktop refresh motion with a future re-entry date.
- **Disqualified**
  - Non-Microsoft account intentionally removed from active work with a reason code.

## Object structure

Each scenario contains:

1. `record`: the core prospect record with stage, cadence fields, activities, review notes, AI summary, and scorecard.
2. `accountProfile`: regional context, current stack, Microsoft workload priority, and sales motion.
3. `handoffPlan`: the intended path from SDR work to later technical or AE ownership.

The catalog is designed to demonstrate queue slicing, stage gates, AI-assist UX, and sales-ops hygiene, not to mirror any real customer data.
