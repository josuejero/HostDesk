# Scenario Catalog

HostDesk ships with 9 seeded scenarios that are copied into MySQL for each user on registration and on `POST /api/demo/reset`.

The source of truth is `data/scenario-catalog.json`. During seeding:

- `record.id` becomes `prospects.external_key`
- record arrays are stored in JSON columns
- stage history gets an initial entry for the seeded stage
- scenarios with `nextTouchDueAt` also receive an open cadence task

## Coverage Summary

The catalog is designed to cover the main demo motions and the operational edge cases that drive queueing and stage discipline:

- 3 primary hero demos
- 6 supporting operational states
- Microsoft motions across Azure Virtual Desktop, Windows 365, Intune, mixed MSP management, nurture, and disqualification cleanup
- every queue view in the app

## Hero Demos

These are the three best first-stop scenarios for demos and screenshots:

### MSP evaluating AVD cost control

- motion: Azure Virtual Desktop
- segment: MSP
- strength: strong AVD cost narrative, reply evidence, clear next step
- good for showing: queue prioritization, scorecard, guided research, and follow-up drafting

### Contractor fleet for Windows 365

- motion: Windows 365
- segment: mid-market services
- strength: fresh inbound lead with clear BYOD and contractor pain
- good for showing: new lead intake, first-touch planning, and queue movement

### Intune compliance rollout

- motion: Intune
- segment: healthcare
- strength: active compliance and app deployment motion with audit urgency
- good for showing: playbook matching, follow-up due workflow, and metrics impact

## Operational State Coverage

The supporting scenarios are not filler. Each one exists to exercise a specific workflow or guardrail:

- `lead-citrix-research`: incomplete record hygiene and blocked handoff
- `lead-healthcare-stale`: stale follow-up detection
- `lead-education-meeting`: valid meeting-booked progression
- `lead-msp-handoff`: valid handoff-ready progression
- `lead-nurture-budget`: intentional nurture timing
- `lead-disqualified`: clean removal from active pipeline with a reason code
