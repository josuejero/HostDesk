# Security Policy

## Supported Versions

HostDesk currently supports the latest commit on the default branch. Versioned release support will be documented here once stable releases are published.

## Reporting a Vulnerability

Please do not open a public GitHub issue for suspected security problems.

Use GitHub private vulnerability reporting if it is available for this repository, or contact the maintainer directly through the repository owner profile.

## Response Timeline

I will try to acknowledge valid reports within 7 days.

For confirmed issues, I will document the affected area, remediation plan, and expected fix timeline. Public disclosure should wait until a fix or mitigation is available unless there is active exploitation.

## Scope

This project is a portfolio/demo application. Reports about authentication, authorization, session handling, CSRF protection, SQL/database access, dependency vulnerabilities, CI/CD workflows, and exposed secrets are in scope.
